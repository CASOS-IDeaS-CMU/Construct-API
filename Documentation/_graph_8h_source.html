204class CONSTRUCT_LIB Graph : public Graph_Interface {
205 //@python->end_include
206 friend class GraphManager;
207 /*
208 While Graph is a child of Graph_Interface, it is the parent to the data structures that actually store the data.
209 These are the DenseDense, DenseSparse, SparseDense, and SparseSparse with their names indicating representation of the row and col dimensions.
210 Trying to include all this in Graph will cause every operation to create two branches in order to determine which representation is being used.
211 By creating child classes that inheriently know what they're represenatation is and assigning the graph pointer to that class,
212 we skip any required branching by making that desicion during construction time.
213 Thus any method of accessing the data has to be a virtual function in Graph.
214 In addition all the iterators for Graph must do any data access including incrementing by calling a member of Graph.
215 This will reroute the function call to whatever represenation was used.
216
217 All the virtual functions use a macro below which simply raises an assertion and then attempts to return a dereferenced null pointer.
218 The dereferenced null pointer is never returned because of the assertion so no unhandled exceptions arise from this.
219 This is purely to ensure the compiler doesn't complain about missing a return value.
220 Reaching these assertions means a child function was not defined to supercede the virtual function.
221 */
