Detailed Description

An interaction model which recreates a social media feed to dictate interactions.

This model uses the media_event data structure to represent and track the event tree/cascade. Agents create events which are stored in Social_Media::list_of_events. These events can be created sponateously or in response to reading an event. What actions are taken depends on the media_user class and any derived from media_user. Future developers may wish to modify the underlying structure of this model. For that reason, a set of virtual functions have been implemented for modifying follower recommendations, unfollowing users, updating feeds, and constructing InteractionMessagess based on read events.

Follower recommendations are handled by Social_Media::add_followees follows the methodology of recommending users based on the similarity of follower bases. If two agents have similar followers and the ego agent is following one of them, the other is recommended to the ego agent to follow. Stronger similarity causes agents to be first in line to be asked whether the ego agent wants to follow them. The number of agents recommended is represented by media_user::pdaf. Unfollowing users is handled by Social_Media::remove_followees where an agent will unfollow an alter agent they currently follow based on reciprocity of the following relationship, the alter agent respondind to the ego agent's events, and a Jaccard Similarity of the ego and alter agent's follower base. The probability to unfollow is weighted by Social_Media::dt / media_user::rf to ensure consistent behaviour independent of dt.

Updating feeds is handled by Social_Media::update_feeds and dictate how the feeds are ordered. New events are added to the feed and old/read event are removed. An event is old when Social_Media::time * Social_Media::dt - media_event::last_used > Social_Media::age. Each users feeds are initially structured by events that mention the user and responses to the user, events of their followees, and all other events. Each of these sub-categories are sorted by their score which is equal to media_event::time_stamp * media_event::child_size. Finally 10% of the entire feed is stochastically swapped to sprinkle new content into the front of a users feed.

All information that is to be conveyed in messages is handled by Social_Media::append_message. Because additional information may added to events than what is currently developed, this will allow developers to add the additional information in the messages being passed around. This function only appends the message items and does not create the message that will be added to the message queue. The message sender is the author of the event being read, not the author of the event being submitted, though sometimes they can be the same index. Because of this the alter index should be included

Gives each user recommended users to follow.

Only agents who are active in the active time network will decide whether to follow any agents they currently aren't following. Changes to follower network only occur after both this function and Social_Media::remove_followees are called. This ensures that there is no impact from the order in which users decide to follow. The descision for whether an agent follows another is contained in media_user::follower_user.

◆ update_feeds()

void Social_Media::update_feeds ( void  )
virtual

Updates each user's feeds.

New events created during the time step are added to each user's feeds while discarding read events. Events are ordered by direct replies or mentions, events of followers, and all other events. Within each category, events are sorted based on media_event::score which is set to media_event::child_size * media_event::time_stamp. After the events have been organized, stochastic shuffling is done on 10% of the feed to avoid a fully deterministic feed.
