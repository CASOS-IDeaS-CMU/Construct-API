template<typename T >
Type_Interface< T > dynet::convert ( T  data)

Wrapper for making template conversions simpler and easier to understand.

This function is intended to be immediately type cast after being called. This allows easy casting between non-standard data types like strings in templates. See example for clarity.

Parameters
dataA data type that is stored for conversion after finishing the function call.
Returns
A Type_Interface that can be type cast into bool, integer, unsigned integer, float, or string.

Example

int main()
{
try {
std::cout << (bool)convert(1.6f) << std::endl;
std::cout << (unsigned int)convert(2) << std::endl;
std::cout << (std::string)convert(true) << std::endl;
std::cout << (unsigned int)convert(-1.4f) << std::endl;
}
catch (std::string e) {
std::cout << "Error: " << e << std::endl;
}
}

Exception Safety

Among the types boolean, integer, and float, no exceptions are thrown. When converting to an unsigned int, similar behavior is expected except an exception is thrown if the original value is negative. When converting a data type to a string, std::to_string is used except for boolean which converts true to "true" and false to "false". When converting a string to a data type, std::stoi and std::stof is used to convert to int, unsigned int and float and a string exception is thrown if either throws an exception. Specifically for boolean, strings of "1" and "true" convert to true, "0" and "false" convert to false, and a string exception is thrown if the string does not match the previous cases. The exception that is thrown is "Could not convert value to data_type" where data_type is a string based on the data type of the data being converted.

CONSTRUCT_LIB std::string dynet::get_type_name ( void  )

Returns a string based on the submitted type.

The function has a type specialization for boolean, integer, unsigned integer, float, and string along with vector<T> and map<unsigned int, T> specializations using the previous 5 specializations. The specific specilization determines which string is returned.

Returns
Returns a string based on the type specialization used. See Example.
